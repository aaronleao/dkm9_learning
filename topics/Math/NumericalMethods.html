Beginner Level

What numerical methods are: most equations arising in practice cannot be solved with formulas. Numerical analysis designs algorithms that approximate the answer to any desired accuracy — and proves how fast they get there.

Root Finding: the Bisection Method

Algorithm (find a root of continuous \(f\) with \(f(a)\cdot f(b) < 0\)):
1. Compute the midpoint \(m = (a+b)/2\).
2. If \(f(a) \cdot f(m) < 0\), the root is in \([a, m]\): set \(b = m\). Otherwise set \(a = m\).
3. Repeat until \(|b - a| < \) tolerance. The root is trapped forever.

Geometric Picture

The root is an animal trapped in a cage \([a, b]\); the sign change is the proof it's inside (by the Intermediate Value Theorem — this is why continuity is required). Each step builds a wall at the midpoint and keeps the half that still has the sign change. The cage halves forever: after \(n\) steps its size is \((b-a)/2^n\). The algorithm is a constructive proof of the IVT.

Interactive: trap the root of \(f(x) = x^3 - x - 2\). Each Step halves the shaded cage; the discarded half flashes red. Exact root ≈ 1.5214.

Root Finding: Newton–Raphson

Algorithm:
1. Start with a guess \(x_0\).
2. Iterate \(x_{n+1} = x_n - \dfrac{f(x_n)}{f'(x_n)}\).
3. Stop when \(|x_{n+1} - x_n| <\) tolerance.

Derivation from the tangent line: the tangent at \(x_n\) is \(y = f(x_n) + f'(x_n)(x - x_n)\). Setting \(y = 0\) (where the tangent crosses zero) and solving for \(x\) gives exactly the update rule.

Geometric Picture

Stand on the curve at \((x_n, f(x_n))\). The tangent is the best straight-line impersonation of the curve; ride it down to where it crosses the axis and call that \(x_{n+1}\). If the curve were exactly its tangent, one step would finish — quadratic convergence (error roughly squares each step) measures how good the impersonation is. Danger: where \(f' \approx 0\) the tangent is nearly flat and the slide shoots you far away — divergence.

Interactive: ride the tangents on \(f(x) = x^3 - 2x + 2\). With \(x_0\) near 0, Newton falls into a 2-cycle (0 → 1 → 0 → …) and never converges — move \(x_0\) below −1 and it homes in on the true root ≈ −1.7693 in a few steps. Old tangents stay faint.

Worked: hardware square roots. \(\sqrt a\) is the positive root of \(f(x) = x^2 - a\). Newton: \(x_{n+1} = x_n - \frac{x_n^2 - a}{2x_n} = \frac12\left(x_n + \frac{a}{x_n}\right)\) — the ancient Babylonian method. For \(a = 2\), \(x_0 = 1\): 1 → 1.5 → 1.41667 → 1.4142157 → 1.41421356… Digits double every step (quadratic convergence).

Exercises:

  1. Do two Newton steps for \(f(x) = x^2 - 2\) from \(x_0 = 2\).
    Solution \(x_1 = \frac12(2 + \frac22) = 1.5\); \(x_2 = \frac12(1.5 + \frac{2}{1.5}) = 1.41\overline{6}\).
  2. How many bisection steps to reach tolerance \(10^{-3}\) from \([0, 8]\)?
    Solution \(8/2^n < 10^{-3} \Rightarrow 2^n > 8000 \Rightarrow n = 13\).

Intermediate Level

Interpolation & Least Squares

Interpolation threads a curve exactly through data points; least squares threads a simple curve near noisy points. Lagrange's formula for \(n{+}1\) points:

\[p(x) = \sum_{i=0}^{n} y_i\,\ell_i(x), \qquad \ell_i(x) = \prod_{j \neq i}\frac{x - x_j}{x_i - x_j}\]

Geometric Picture

Each basis polynomial \(\ell_i\) is a switch: it equals 1 at its own node \(x_i\) and 0 at every other node. The interpolant is a superposition — each data point broadcasts its own bump, and at each node all the other bumps are silent. For least squares: each data point pays a penalty equal to the actual square built on its vertical miss; fitting means tilting the line until the total square area is smallest ("normal equations" = the residual ends up perpendicular to the model space).

Interactive: drag the four y-values. Mode 0 shows the interpolating cubic through all points (computed by solving the Vandermonde system with solveLinear); mode 1 shows the least-squares line with each residual drawn as a literal square — adjust and watch the total area; the dashed line is the optimal fit.

Step-by-step: Lagrange through (0, 1), (1, 3), (2, 2)

Step 1: Build the three switches
\(\ell_0 = \frac{(x-1)(x-2)}{(0-1)(0-2)} = \frac{(x-1)(x-2)}{2}\),   \(\ell_1 = \frac{x(x-2)}{1(1-2)} = -x(x-2)\),   \(\ell_2 = \frac{x(x-1)}{2(2-1)} = \frac{x(x-1)}{2}\)

Step 2: Superpose with the data as weights \[p(x) = 1\cdot\ell_0 + 3\cdot\ell_1 + 2\cdot\ell_2 = -\tfrac32 x^2 + \tfrac72 x + 1\] Step 3: Verify — \(p(0)=1\) ✓, \(p(1)=3\) ✓, \(p(2)=2\) ✓.
Step-by-step: normal equations for the least-squares line

Minimize \(S(m, b) = \sum_i (y_i - mx_i - b)^2\). Set both partials to zero (calculus meets linear algebra):
\(\frac{\partial S}{\partial b} = -2\sum(y_i - mx_i - b) = 0\) → \(nb + m\sum x_i = \sum y_i\)
\(\frac{\partial S}{\partial m} = -2\sum x_i(y_i - mx_i - b) = 0\) → \(b\sum x_i + m\sum x_i^2 = \sum x_i y_i\)
Two linear equations in \((b, m)\) — solved in the widget by solveLinear.

Runge's warning: interpolating many equally spaced points with one high-degree polynomial makes the "switches" ring violently near the interval ends (Runge phenomenon). Practice uses piecewise cubics (splines) — the technology behind fonts and CAD — or Chebyshev-spaced nodes.

Real-World Applications: sensor calibration curves, financial trendlines, font rendering (spline interpolation of glyph outlines), GPS track smoothing.

Numerical Integration

Approximating \(\int_a^b f\,dx\) with ever-better "bricks": rectangles (midpoint), tilted-roof trapezoids, and parabolic roof tiles (Simpson). Error orders per panel width \(h\): midpoint & trapezoid \(O(h^2)\), Simpson \(O(h^4)\).

Interactive: four methods on \(\int_0^1 x^2 dx = \frac13\). Same \(n\), very different errors — Simpson's parabolas hug the curve so well the error is invisible almost immediately.

Worked, n = 4 on \(\int_0^1 x^2dx\):
Trapezoid: \(\frac{h}{2}[f(0) + 2f(\frac14) + 2f(\frac12) + 2f(\frac34) + f(1)] = \frac{1}{8}[0 + \frac18 + \frac12 + \frac98 + 1] = 0.34375\) (error 1.04×10⁻²).
Simpson: exact for polynomials up to degree 3, so it returns \(\frac13\) exactly — error 0. That's the \(O(h^4)\) advantage.
Real-World Applications: distance from sampled GPS speed, probabilities from density functions, energy from power meters — any "area under sampled data" question.

Advanced Level

ODE Solvers: Euler & Runge–Kutta

An initial value problem \(y' = f(x, y),\ y(x_0) = y_0\) asks: given the local slope rule, reconstruct the path.

Euler's method (algorithm): \(y_{n+1} = y_n + h\,f(x_n, y_n)\) — take the current slope, walk straight for one step \(h\). Error \(O(h)\).

RK4 (algorithm): sample the slope four times per step — \(k_1\) at the start; \(k_2, k_3\) at the midpoint (using \(k_1\), then \(k_2\)); \(k_4\) at the end (using \(k_3\)) — then walk the weighted average: \[y_{n+1} = y_n + \tfrac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4)\] Error \(O(h^4)\): halving \(h\) cuts the error 16-fold.

Geometric Picture

The ODE paints a slope field — a tiny compass needle at every point of the plane (drawn in the widget's background). Euler is a hiker who checks the compass only every \(h\) kilometers and walks straight between checks: on curved paths he systematically cuts corners and drifts. RK4 is the smarter hiker who scouts: he checks the compass at the start, twice at the midpoint, and at the far end, then walks the averaged direction. Four compass checks per step buy four orders of accuracy.

Interactive: \(y' = y,\ y(0) = 1\) (exact: \(e^x\), dashed). Step both hikers with the same \(h\) and compare their errors at \(x = 1\) in the readout. Then shrink \(h\) and watch Euler improve linearly, RK4 dramatically.

The energy-drift picture (pendulum): for the frictionless pendulum \(\theta'' = -\sin\theta\), the true orbit in phase space \((\theta, \theta')\) is a closed loop (energy conserved). Euler's corner-cutting systematically gains energy — its orbit spirals outward — while RK4's stays closed for ages. This is why physics engines and orbital mechanics never use plain Euler.

Real-World Applications: game physics engines, spacecraft trajectory planning, epidemic models (SIR), circuit simulation (SPICE integrates ODEs), weather prediction.

Linear Systems Revisited: LU & Conditioning

LU factorization: Gaussian elimination, recorded. Each elimination step is an elementary matrix; collecting them gives \(A = LU\) with \(L\) lower-triangular (the multipliers) and \(U\) upper-triangular (the result). Worked 3×3 (same system as Linear Algebra § Gaussian elimination): the multipliers used there (2, 1, −⅓) drop straight into \[L = \begin{pmatrix}1 & 0 & 0\\ 2 & 1 & 0\\ 1 & -\tfrac13 & 1\end{pmatrix},\quad U = \begin{pmatrix}1 & 1 & 1\\ 0 & -3 & -1\\ 0 & 0 & -\tfrac73\end{pmatrix}\] Once factored, each new right-hand side costs only two triangular solves.

Geometric picture: each elimination step is a shear of space; \(L\) is the record of shears, \(U\) the sheared result.
Jacobi iteration (algorithm): solve row \(i\) for its own diagonal unknown, using the previous iterate for the others: \[x_i^{(k+1)} = \frac{1}{a_{ii}}\Big(b_i - \sum_{j \neq i} a_{ij}x_j^{(k)}\Big)\] Converges if \(A\) is strictly diagonally dominant (each diagonal beats the rest of its row).

Geometric picture (cobweb): in 2D, from the current guess move horizontally onto line 1, then vertically onto line 2, repeat — a staircase spiraling into the intersection when dominance holds, out when it fails.
Conditioning (worked): solve \[\begin{cases} x + y = 2 \\ x + 1.001y = 2.001 \end{cases} \Rightarrow (x, y) = (1, 1)\] Now nudge one data value by 0.1%: right side \(2.001 \to 2.003\) gives \((x, y) = (-1, 3)\). A 0.1% data change moved the answer by 200% — the lines are nearly parallel, so their crossing leaps at the slightest wiggle. The condition number \(\kappa(A) = \|A\|\|A^{-1}\|\) (here ≈ 4000) is exactly this amplification factor. Near-perpendicular lines (\(\kappa \approx 1\)) barely react.

Exercises:

  1. Run two Jacobi iterations on \(4x + y = 9,\ x + 3y = 7\) from \((0, 0)\).
    Solution \(x^{(1)} = 9/4 = 2.25,\ y^{(1)} = 7/3 \approx 2.333\); \(x^{(2)} = (9 - 2.333)/4 \approx 1.667,\ y^{(2)} = (7 - 2.25)/3 \approx 1.583\). Converging to \((2, 5/3)\).
  2. Why is bisection immune to bad conditioning, but Newton is not?
    Solution Bisection uses only signs, which tiny perturbations rarely flip far from the root; Newton divides by \(f'(x_n)\), amplifying errors wherever the tangent is flat.

Research Level

Krylov Subspace Methods

For systems with millions of unknowns (PDE discretizations), even LU is too expensive. Krylov methods (Conjugate Gradient, GMRES) search for the solution inside the growing subspace \(\text{span}\{b, Ab, A^2b, \dots\}\), touching \(A\) only through matrix–vector products — perfect for sparse matrices. Preconditioning (transforming the system to have a friendlier spectrum) is where much of the research lives.

Randomized Numerical Linear Algebra

Random projections can sketch a huge matrix down to a small one while provably preserving its important directions — enabling fast approximate SVD and least squares on data too large to read twice. A very active field connecting numerical analysis with probability and theoretical CS.

Structure-Preserving (Geometric) Integrators

Symplectic integrators (used in planetary simulations) preserve the geometric invariants of Hamiltonian systems — the reason million-year solar-system simulations don't spiral apart like the Euler pendulum. Designing integrators that respect energy, symplectic form, or constraints exactly is modern numerical analysis at its most geometric.
